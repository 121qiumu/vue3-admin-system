# 源码学习导航

这份文档是配合源码里的“学习注释”一起看的。
如果你刚接触这个项目，最推荐的方式不是随机点文件，而是顺着启动链往下读。

## 推荐阅读顺序

1. `src/main.js`
2. `src/App.vue`
3. `src/plugins/index.js`
4. `src/store/index.js`
5. `src/router/index.js`
6. `src/permission/index.js`
7. `src/permission/guard.js`
8. `src/request/index.js`
9. `src/locales/helper.js`
10. `src/layout/index.vue`
11. `src/views/...`

## 最核心的一条主线

`main.js -> App.vue -> setupPlugins(app) -> store/router/i18n/permission -> layout -> views`

你可以把它理解成下面这个流程：

1. `main.js` 创建 Vue 应用实例。
2. `App.vue` 给整个应用包上一层全局能力。
3. `plugins/index.js` 安装 Pinia、i18n、Router、权限守卫、指令。
4. `store/index.js` 恢复 token、主题、语言、标签页等缓存状态。
5. `router/index.js` 先注册静态路由。
6. `permission/index.js` 和 `permission/guard.js` 决定什么时候把动态路由再注入进去。
7. 命中 `Layout` 后，再渲染左侧布局、顶部布局或混合布局。
8. 最后才进入具体业务页面，比如用户管理、角色管理、菜单管理、Dashboard。

## 你最容易卡住的几个点

### 为什么有些文件只 export，不直接干活

因为这个项目是分层的：

- `src/api/*.js` 对外导出接口函数。
- `src/store/modules/*.js` 对外导出 store。
- `src/hooks/*.js` 对外导出组合式函数。
- `src/router/modules/*.js` 对外导出路由配置对象。
- `src/views/*.vue` 使用 `<script setup>`，组件导出是自动完成的。

### 为什么路由有静态和动态两套

- 静态路由：项目一启动就要有，比如登录页、404、首页。
- 动态路由：要等拿到权限码后，才能决定当前账号能不能看到，比如系统管理模块。

### 为什么 store 要做缓存恢复

因为页面刷新后，内存状态会丢失。
如果不恢复：

- token 没了，看起来像退出登录。
- 主题没了，页面风格重置。
- 语言没了，文案切回默认值。
- 标签页没了，导航体验会断。

### 为什么国际化会和路由绑在一起

这个项目采用语言前缀路由，比如：

- `/zh-CN/dashboard`
- `/en-GB/dashboard`

所以语言不是单独保存在某个变量里，而是直接进入 URL。
相关关键文件是：

- `src/locales/resolve.js`
- `src/locales/helper.js`
- `src/plugins/i18n.js`

## 这次我加的注释主要分两层

1. 文件头学习注释：几乎每个源码文件都有，告诉你这个文件是干什么的、依赖谁、导出什么、通常被谁引用。
2. 核心骨架逐段注释：重点放在启动顺序、权限恢复、请求拦截、语言路径、布局壳这些最容易绕晕的地方。

## 这次没有去动的目录

为了避免把第三方代码和构建产物也注释得很乱，没有处理这些目录：

- `node_modules`
- `dist`
- `.git`
- `.pnpm-store`

另外像 `package.json`、`jsconfig.json` 这种 JSON 文件本身不支持真正的代码注释，所以说明尽量放到了源码文件和这份文档里。
